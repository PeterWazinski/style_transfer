# Built-in Style Chains — Implementation Plan

## 1. Summary

Add a second tab to the left-panel dock that shows curated built-in style chains.
Built-in chains ship with the app and are managed at development time via a notebook.
The implementation mirrors the existing styles system: catalog JSON + registry class + gallery view.
This feature also takes the opportunity to improve the existing `StyleGalleryView` with tooltips and
a right-click context menu.

---

## 2. Decisions

| # | Question | Decision |
|---|---|---|
| Q1 | **Apply mode from gallery** | Double-click = apply fresh (reset + run); Re-apply button = apply chain on top of current photo state. Both styles and chains get a **right-click context menu**: styles offer *Apply* + *Re-Apply*; chains offer *Apply* + *Append*. |
| Q2 | **Preview mandatory?** | Yes. `add_style_chain.ipynb` generates the preview automatically on `sample_images\arch.png` and aborts if preview generation fails. |
| Q3 | **Preview resolution** | 256 px (same as styles). Can be raised to 512 px later if thumbnails look poor. |
| Q4 | **`sample_images/style-chains/*.yml`** | **Not migrated.** A new `style_chains/` directory will be populated with freshly curated chains via the notebook. `sample_images/style-chains/` stays as-is for `--style-chain-overview` use. |
| Q5 | **Startup validation** | Non-blocking: grey-out invalid chains + ⚠ badge + `logger.warning`. No dialog unless *all* chains are broken. |
| Q6 | **Catalog key** | `{"chains": [...]}` |
| Q7 | **`style_chains/` layout** | One subdir per chain: `style_chains/<id>/chain.yml` + `style_chains/<id>/preview.jpg` — consistent with `styles/<id>/`. |
| Q8 | **BatchStyler auto-discovery** | Out of scope for this iteration. |

---

## 3. Confirmed Improvements

1. **Tooltip — chains**: hover over a chain thumbnail shows the ordered step list (style name + strength %).
2. **Tooltip — styles**: hover over a style thumbnail shows the `description` string from `styles/catalog.json`.
3. **`tags` field**: added to both `BuiltinChainModel` (backend only) and `StyleModel` (backend only). No UI logic; reserved for future filtering.
4. **`step_count`** denormalized in `style_chains/catalog.json` — gallery card can show "3 steps" without loading the YAML.
5. **Warning badge** — translucent grey overlay + ⚠ icon via `ThumbnailDelegate` extension for invalid chains.
6. **`add_style_chain.ipynb` validates styles** before writing — every referenced style must exist in `styles/catalog.json`; the notebook aborts if any are missing.
7. **Right-click context menu** for both `StyleGalleryView` and `ChainGalleryView` (see Q1).
8. **`training/add_style_chain.ipynb`** (not `scripts/`) — consistent with existing add-style notebooks.

## 5. Data Model

### 5.1 `style_chains/catalog.json`

```json
{
  "chains": [
    {
      "id": "abstract-color",
      "name": "Abstract Color",
      "description": "Cubism base with abstract colour overlay.",
      "chain_path": "style_chains/abstract-color/chain.yml",
      "preview_path": "style_chains/abstract-color/preview.jpg",
      "step_count": 2,
      "tags": ["abstract", "colorful"]
    }
  ]
}
```

### 5.2 `BuiltinChainModel` dataclass (`src/core/chain_models.py`)

```python
@dataclass
class BuiltinChainModel:
    id: str
    name: str
    chain_path: str          # relative to repo root
    preview_path: str = ""
    description: str = ""
    step_count: int = 0
    tags: list[str] = field(default_factory=list)

    def chain_path_resolved(self, root: Path) -> Path: ...
    def preview_path_resolved(self, root: Path) -> Path: ...
    def to_dict(self) -> dict: ...
    @classmethod
    def from_dict(cls, data: dict) -> "BuiltinChainModel": ...
```

### 5.3 `ChainStore` (`src/core/chain_models.py`)

Read-only at runtime (notebook writes the catalog):

```python
class ChainStore:
    def __init__(self, catalog_path: Path) -> None: ...
    def load(self) -> list[BuiltinChainModel]: ...
```

### 5.4 `StyleModel` addition (`src/core/models.py`)

Add one new field (no migration needed — `from_dict` ignores unknown keys already):

```python
tags: list[str] = field(default_factory=list)   # reserved for future filtering
```

## 6. Registry (`src/core/chain_registry.py`)

```python
class ChainNotFoundError(KeyError): ...

class BuiltinChainRegistry:
    def __init__(self, catalog_path: Path) -> None: ...
    def list_chains(self) -> list[BuiltinChainModel]: ...   # sorted by name
    def get(self, chain_id: str) -> BuiltinChainModel: ...
    def validate_styles(self, registry: StyleRegistry) -> dict[str, list[str]]:
        """Return {chain_id: [missing_style_name, ...]} for every broken chain."""
```

`validate_styles` result is passed to `ChainGalleryView` to drive the badge/grey-out logic.  
No mutating methods — registry is read-only at runtime.

## 7. UI Changes

### 7.1 Dock widget: tabs

Replace `StyleGalleryView` as direct child of the dock with a `QTabWidget`:

```
┌─ Styles ─┬─ Chains ─┐
│ [thumb]  │ [thumb]  │
│ [thumb]  │ [thumb]  │
└──────────┴──────────┘
```

### 7.2 `StyleGalleryView` improvements

- **Tooltip**: `item.setToolTip(style.description)` when `description` is non-empty.
- **Right-click context menu** (replaces the existing noop `CustomContextMenu`):
  - *Apply* — same as double-click (emit `style_apply_requested`)
  - *Re-Apply* — new signal `style_reapply_requested(StyleModel)`

### 7.3 `ChainGalleryView` (`src/stylist/chain_gallery.py`)

- `QListView` in `IconMode`, same `ThumbnailDelegate`.
- Items carry `BuiltinChainModel` as `Qt.UserRole`.
- Invalid chains (from `validate_styles`) rendered grey + ⚠ overlay.
- **Tooltip**: lazily loads `chain.yml` on first hover; formats the step list as plain text.
- Signals:
  - `chain_selected(BuiltinChainModel)`
  - `chain_apply_requested(BuiltinChainModel)` — double-click → apply fresh
  - `chain_append_requested(BuiltinChainModel)` — right-click → Append
- **Right-click context menu**:
  - *Apply* — emit `chain_apply_requested`
  - *Append* — emit `chain_append_requested`

### 7.4 `ChainGalleryController` mixin (`src/stylist/chain_gallery_controller.py`)

```python
def _apply_builtin_chain(self: "MainWindow", chain: BuiltinChainModel) -> None:
    """Reset photo state then run chain steps (apply fresh)."""

def _append_builtin_chain(self: "MainWindow", chain: BuiltinChainModel) -> None:
    """Apply chain on top of current photo state (delegates to _append_style_chain logic)."""
```

Both resolve `chain.chain_path_resolved(root)`, parse it with `load_style_chain`, then delegate
to the existing step-loop in `StyleChainController`.

## 8. Startup Validation

In `src/stylist/app.py`:

```python
_CHAIN_CATALOG_PATH: Path = _project_root() / "style_chains" / "catalog.json"

# in main():
chain_registry = BuiltinChainRegistry(catalog_path=_CHAIN_CATALOG_PATH)
invalid_chains = chain_registry.validate_styles(registry)   # dict[id, [missing...]]
for cid, missing in invalid_chains.items():
    logger.warning("Built-in chain '%s' references missing styles: %s", cid, missing)
```

`invalid_chains` keys are forwarded to `ChainGalleryView` as a `set[str]` for badge rendering.

## 9. Development Notebook (`training/add_style_chain.ipynb`)

| Step | Action |
|---|---|
| 1 | Setup: locate repo, load `style_chains/catalog.json`, list existing IDs |
| 2 | Author the chain YAML (steps + strengths) |
| 3 | **Validate styles** — abort if any referenced style is absent from `styles/catalog.json` |
| 4 | Generate preview by applying the chain to `sample_images/arch.png` at 256 px |
| 5 | Copy files to `style_chains/<id>/` and append entry to catalog |
| 6 | Smoke check: reload catalog, confirm entry present, display thumbnail |

Backend helper `training/add_style_chain_helper.py`:
- `setup()` → `ChainCatalogContext`
- `validate_chain_styles(chain: StyleChain, styles_catalog: dict) -> list[str]`
- `install_chain(*, chain_yml, name, description, tags, catalog_path, ...)` → writes + updates catalog
